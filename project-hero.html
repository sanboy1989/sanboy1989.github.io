Context & Problem

The last-mile problem nobody solved for C2C

C2C 市場無人解決的末端配送問題

In Hong Kong's C2C secondhand market, "meet in-person" is the dominant exchange method. But most sellers are across town. Buyers don't want to pay courier fees for a $50 item. Sellers don't want to make a special trip. The transaction dies before it starts.

在香港二手交易市場，「親身交收」是主流方式。但大多數賣家在城市另一端。買家不願為 50 元物品支付快遞費。賣家不想專程跑一趟。交易在開始前就已死亡。

On the Way Hero reframes this. Someone is already commuting from Kowloon to Hong Kong Island every morning. That empty bag space is idle logistics capacity. This platform matches idle commuters ("Heroes") with buyers and sellers whose pickup points fall along the Hero's existing route — turning dead time into micro-earnings.

On the Way Hero 重新框架了這個問題。每天早上已有人從九龍通勤到港島。那個空背包就是閒置的物流容量。本平台將空閒的通勤者（「英雄」）與取貨地點恰好在其路線上的買賣家配對——把死時間轉化為微收入。

Tech Deep Dive

The Stack & the Hard Parts

技術棧與解決難題

Vue.js as a PWA means users can install it to their home screen with no App Store friction. Google Maps API powers the route polyline rendering and proximity filtering. Service Workers cache the map tiles and active delivery state so the app stays usable on an MTR dead zone.

Vue.js 作為 PWA 意味著用戶無需通過應用商店即可安裝到主螢幕。Google Maps API 驅動路線多邊形渲染和鄰近過濾。Service Workers 緩存地圖磁貼和活躍配送狀態，讓 App 在港鐵無信號區依然可用。

Key Design Decision
關鍵設計決策
Fuzzy geolocation to protect privacy
模糊定位保護隱私

Showing a Hero's exact real-time location to strangers is a non-starter from a safety perspective. Before a delivery is confirmed, the platform only exposes an approximate area (500m radius) — enough for route matching, not enough to track someone. Exact coordinates unlock only after both parties confirm.

向陌生人展示英雄的精確實時位置，從安全角度來看是不可行的。在配送確認之前，平台只暴露大致區域（500 米半徑）——足以匹配路線，卻不足以追蹤某人。精確坐標僅在雙方確認後才解鎖。

Hardest Bug
最難解決的 Bug

Route matching was returning false negatives — Heroes who were clearly on the right route were being filtered out. The root cause was coordinate precision drift in the Maps API: the snapped-to-road route coordinates differed by up to 80m from the raw GPS points users submitted. Fixed by increasing the corridor tolerance and switching from point-in-radius checks to polyline buffer intersections, which is far more robust to GPS noise.

路線匹配出現假陰性——明顯在正確路線上的英雄被過濾掉了。根本原因是 Maps API 中的坐標精度漂移：道路捕捉後的路線坐標與用戶提交的原始 GPS 點相差最多 80 米。解決方案是增加走廊容忍度，並從半徑點查詢切換為多段線緩衝區交集，這對 GPS 噪聲的容錯性要高得多。
